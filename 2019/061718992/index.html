环境准备

需要准备gcc，redis，ruby，zlib，rubygems，openssl，redis-4.1.0.gem等相关依赖。

说明：redis5.0版本以上，可以使用官方自带的集群构建方法，只需要安装gcc，redis即可，查看标*章节即可。较低版本需要ruby构建redis集群，除gcc，redis之外，还需要安装ruby，zlib，rubygems，openssl，redis.gem等。

检查gcc环境

输入命令gcc -v，如果显示下面内容，说明gcc已经安装。

[root@localhost opt]# gcc -v
使用内建 specs。
COLLECT_GCC=gcc
COLLECT_LTO_WRAPPER=/usr/libexec/gcc/x86_64-redhat-linux/4.8.5/lto-wrapper
目标：x86_64-redhat-linux
配置为：此处省略部分内容...
gcc 版本 4.8.5 20150623 (Red Hat 4.8.5-16) (GCC)

手动安装gcc环境

如果未显示上述内容，则需要下载相关依赖，进行手动安装。下列为相关依赖列表：

mpfr-3.1.1-4.el7.x86_64.rpm
libmpc-1.0.1-3.el7.x86_64.rpm
kernel-headers-3.10.0-123.el7.x86_64.rpm
glibc-headers-2.17-55.el7.x86_64.rpm
glibc-devel-2.17-55.el7.x86_64.rpm
cpp-4.8.2-16.el7.x86_64.rpm
gcc-4.8.2-16.el7.x86_64.rpm

注：这里提供一个rmp下载地址：传送门，复制上面的rpm包名，直接搜索下载。

或者直接下载：mpfr, libmpc, kernel-headers, glibc-headers, glibc-devel, cpp, gcc

将下载好的rmp包上传到服务器/opt/redis-cluster/package/gcc4.8.2[路径可自定义]路径下，执行统一安装命令rpm -Uvh *.rpm --nodeps --force进行安装。

[root@localhost gcc4.8.2]# rpm -Uvh *.rpm --nodeps --force

使用gcc -v命令验证安装是否成功。

安装依赖

redis5.0以上，只需要下载redis即可。5.0以下版本需要下载ruby等依赖文件。

下载依赖文件

安装前需要下载redis，ruby，zlib，rubygems，openssl，redis-4.1.0.gem等相关依赖。

  • redis5.0.5下载：传送门

  • ruby2.5.5下载：传送门

  • zlib1.2.11下载：传送门

  • rubygems3.0.3下载：传送门

  • openssl1.1.0i下载：传送门

  • redis-rubygems4.1.0下载：传送门

直接下载方式：redis5.0.5, ruby2.5.5, zlib1.2.11, rubygems3.0.3, openssl1.1.0i, redis-rubygems4.1.0

安装

下载好相关依赖文件后，利用xftp工具将依赖文件上传到/opt/redis-cluster/package[路径可自定义]，下面开始安装各个依赖。

安装ruby2.5.5

解压ruby：tar -zxvf ruby-2.5.5.tar.gz

[root@localhost package]# cd /opt/redis-cluster/package
[root@localhost package]# tar -zxvf ruby-2.5.5.tar.gz       # 解压

编译安装ruby，其中./configure --prefix=/usr/local/ruby命令中，--prefix参数将ruby安装到指定目录，也可以自定义路径。

[root@localhost ruby-2.5.5]# cd /opt/redis-cluster/package/ruby-2.5.5 # 进入到ruby源码文件夹
[root@localhost ruby-2.5.5]# ./configure --prefix=/usr/local/ruby
[root@localhost ruby-2.5.5]# make && make install          # 编译安装

配置ruby环境变量：

编辑/etc/profile配置文件，在PATH中添加ruby的bin目录路径。执行source /etc/profile命令使配置立即生效。

[root@localhost ruby]# vim /etc/profile           # 编辑profile配置文件
[root@localhost ruby]# source /etc/profile        # 使配置文件立即生效
[root@localhost ruby]# echo $PATH                 # 输出PATH信息
/usr/local/ruby/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/root/bin
[root@localhost ruby]# ruby -v                    # 查看ruby版本信息
ruby 2.5.5p157 (2019-03-15 revision 67260) [x86_64-linux]
安装zlib1.2.11

进入/opt/redis-cluster/package/文件夹，解压并编译安装zlib。

[root@localhost package]# cd /opt/redis-cluster/package/            # 进入zlib文件夹
[root@localhost package]# tar -zxvf zlib-1.2.11.tar.gz              # 解压zlib文件
[root@localhost package]# cd zlib-1.2.11/
[root@localhost zlib-1.2.11]# ./configure --prefix=/usr/local/zlib
[root@localhost zlib-1.2.11]# make && make install                  # 编译安装zlib
安装rubygems3.0.3

RubyGems 是 Ruby 的一个包管理器，它提供一个分发 Ruby 程序和库的标准格式，还提供一个管理程序包安装的工具。

[root@localhost package]# cd /opt/redis-cluster/package/
[root@localhost package]# tar -zxvf rubygems-3.0.4.tgz
[root@localhost package]# cd rubygems-3.0.4/
[root@localhost rubygems-3.0.4]# ruby setup.rb

可能是ruby在编译时，未编译zlib依赖，可重新编译zlib模块：

cd /opt/redis-cluster/package/ruby-2.5.5/ext/zlib，进入到ruby/zlib模块路径。编译zlib过程中，需要修改Makefile文件：

280行：#zlib.o: $(top_srcdir)/include/ruby.h 注释掉本行，替换为下一行的内容
修改为：zlib.o: ../../include/ruby.h
[root@localhost zlib]# pwd

[root@localhost zlib]# make install
/usr/bin/install -c -m 0755 zlib.so /usr/local/ruby/lib/ruby/site_ruby/2.5.0/x86_64-linux

解决完缺少zlib的问题后，重新安装rubygems。

[root@localhost package]# cd /opt/redis-cluster/package/
[root@localhost package]# tar -zxvf rubygems-3.0.4.tgz    # 解压rubygems压缩包
[root@localhost package]# cd rubygems-3.0.4/
[root@localhost rubygems-3.0.4]# ruby setup.rb            # 安装rubygems
[root@localhost rubygems-3.0.4]# 
[root@localhost rubygems-3.0.4]# gem -v                   # 查看rubygems版本号
3.0.4

安装完成后，配置rubygems环境变量：

rubygems安装完成后，配置rubygems环境变量。编辑vim /etc/profile文件，在文件当中PATH当中加入安装rubygems的bin目录路径，路径为安装目录bin文件夹/root/public/rubygems-3.0.3/bin/ 【这个路径就是你解压出rubygems的路径】 然后保存，执行命令：source /etc/profile让配置立即生效，然后echo $PATH可以查看变量已被添加。

安装openssl1.1.0i

解压openssl，编译安装

[root@localhost package]# tar -zxvf openssl-1.1.0i.tar.gz   ## 解压openssl
[root@localhost package]# cd openssl-1.1.0i
[root@localhost openssl-1.1.0i]# ./config -fPIC --prefix=/usr/local/openssl enable-shared ## 配置openssl
[root@localhost openssl-1.1.0i]# make && make install  ## 编译安装

redis常见错误

错误1

[ERR] Node 192.168.88.111:7000 is not empty. Either the node already knows other nodes (check with CLUSTER NODES) or contains some key in database 0.

错误原因：由于上次redis集群没有配置成功，生成了每个节点的配置文件和db的备份文件，所以才会产生这个错误。

解决方法：需要删除六个redis节点的备份文件，数据库文件，集群配置文件，即每个节点中的appendonly.aof、dump.rdb、node_xxx.conf文件都要被删除。

另外还需要连接各个redis节点，清空数据。

[root@localhost 7000]# redis-cli -c -p 7000   # 连接redis节点
127.0.0.1:7000> flushdb                       # 清空数据库
OK
127.0.0.1:7000> cluster reset                 # 重置
OK

操作完上述步骤，重启所有的redis节点，之后再创建集群即可。

错误2

redis-cluster (error) MOVED 866 192.168.88.111:7000

解决方法：连接时，使用redis-cli -c -h 127.0.0.1 -p 7000，而不是redis-cli -h 127.0.0.1 -p 7000
